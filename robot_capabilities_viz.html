System overview

Turret wrap (robot-centric [−180°, 180°)

Target and current angle are kept in [−180°, 180°). The controller picks the nearest equivalent target in ticks so the turret always rotates the short way and does not spin past ±180° (avoids wire tangling).

Robot-centric angle: −180° ≤ θ < 180° → shortest path

Diagram: turret at 170° (green); target −170° (orange) → moves 20° the other way (yellow arc), not 340°.

Velocity compensation

Virtual goal = goal − velocity × (gain × timeOfFlight). When moving backward, virtual goal is farther than the real goal; when moving sideways, aim is shifted opposite to velocity so the note still lands at the goal.

Green = goal, orange = virtual goal (when moving), yellow = aim line.

AirSort flow

  • Toggle: Operator Back → AirSort on/off; index resets to 0 when turned on.
  • Motif: AprilTag 21 → G,P,P; 22 → P,G,P; 23 → P,P,G. Stored once.
  • Ball at launcher: c3 only (green / purple / unknown).
  • Shot: desired = motif[shotIndex]. Match → fast shot (min T); no match → slow shot (max T).
  • Advance: When ball leaves launcher (c3 empty), schedule advance in 250 ms (tunable), then index 0→1→2.

Vision & AprilTag

  • Goal (20, 24): Lock-on turret track, camera distance (12–200 in), optional reloc (blue 20), rumble when aimed.
  • Obelisk (21, 22, 23): AirSort motif only; decimation 2 when AirSort on.
